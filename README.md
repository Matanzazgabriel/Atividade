# Sistema de Gerenciamento de Ordens de Serviço (Exemplo)

Projeto de exemplo implementando conceitos de POO: encapsulamento, herança, polimorfismo e abstração,
com separação de responsabilidades entre camadas (models, persistence, ui).

## Estrutura de pastas
- models.py      : definição das classes do domínio (Cliente, Tecnico, Produto, OrdemServico, ItemOS)
- persistence.py : funções simples para salvar/carregar JSON (db.json)
- ui.py          : interface mínima (linha de comando) e demo de execução
- main.py        : ponto de entrada

## Bibliotecas utilizadas
- Nenhuma além da biblioteca padrão do Python.

## Como executar
```bash
python main.py
```
